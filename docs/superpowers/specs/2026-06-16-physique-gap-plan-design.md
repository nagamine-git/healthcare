# 理想体型ギャップ → 実践プラン 設計

日付: 2026-06-16 / 状態: 実装・デプロイ済み

## 目的
「この体型になりたいなら結局何をすべきか」を、現在地と目標のギャップ + エネルギー収支から
数学的に逆算し、医学的に正しい優先順位 (食事 > 筋トレ > 有酸素) で処方する。

## 科学的根拠
- 体重変化はエネルギー収支。脂肪 1kg ≈ 7700 kcal。脂肪減はカロリー収支が支配的レバー
  =「食事が主」。同じ赤字を運動で作る非効率を分単位で提示 (シャドーボクシング MET=7)。
- 筋増は抵抗運動が必須刺激。タンパク質 1.6–2.2 g/kg (Morton 2018) が基質。有酸素は
  筋肥大の主因でなく、過度は干渉効果。シャドーボクシングは「楽しい=継続に効く」ので
  コンディショニングとして正しく位置づけ (筋量の主因ではない)。
- 持続可能レート: 脂肪減 0.5–1%/週、筋増 中級 0.25–0.5%/月 (女性は約半分)。
- 同時達成 (recomp) は初心者/体脂肪高めで起きやすい。

## 計算
- BMR = Mifflin-St Jeor (現体重)。TDEE = BMR + Garmin実測活動消費(14日平均)、無ければ BMR×1.45。
- 現在/目標を脂肪量・除脂肪量に分解 → 方向判定 (cut/recomp/lean_bulk/maintain)。
- カロリー目標: cut=赤字(0.75%/週、TDEEの25%上限、BMR下回らない)、recomp=緩い赤字、
  lean_bulk=+10%、maintain=TDEE。マクロ: タンパク質=g/kg×体重、脂質下限0.8g/kg、残り炭水化物。
- 食事vs運動: 赤字を高強度シャドーボクシングで作る所要分/日。
- 寄与率: 方向別のモデル優先度(%)。因果分解は不可能なのでエネルギー数理に基づく実務優先度
  として明示ラベル。
- タイムライン: 脂肪/筋それぞれの所要週、ETA=max。

## API / UI
- `GET /api/physique-plan` (profile router)。`recomposition_plan(app_today())`。
- frontend: 目標タブに `PhysiqueGapPlan` カード (現在→目標、優先度バー、カロリー/マクロ、
  食事vs運動の核心、筋トレ処方、ETA、注記)。

## テスト
純粋数理 (マクロ/寄与率合計100/ETAラベル) + API統合 (recomp方向・赤字・実測フラグ)。
