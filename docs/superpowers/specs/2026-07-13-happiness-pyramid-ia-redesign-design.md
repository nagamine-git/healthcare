# 幸福ピラミッド 情報アーキテクチャ再設計(Apple級シンプル + 全解像度)

日付: 2026-07-13

## 目的

アプリの領域(ライフドメイン)を、幸福研究に準拠した **4本柱ピラミッド**に再編し、ホームは
Apple製品のように**とことんシンプル**(上位カードのみ)にしつつ、**掘れば全解像度**(既存データを
一切捨てない)にする。表面を静かに、中身は温存。

## 科学的根拠

- 幸福研究の一致点: **PERMA**(良い感情/没頭/関係/意味/達成)+ **自己決定理論**(自律・有能・関係)。
- 予防医学の健康土台: 睡眠・栄養・運動・物質(依存)・ストレス。
- **人間関係は幸福・健康の最大の予測因子**(ハーバード成人発達研究)→ 独立本柱に。
- ユーザー当初案の欠落を補正: 睡眠/栄養/体組成/心肺(VO2max)/気分・ストレス/瞑想 を健康に明示。
  頭痛の二重掲載を解消。「心=本/映画/漫画」を「心(気分/ストレス/瞑想)」と「意味・楽しみ(本映画漫画)」に分離。

## 確定タクソノミー(既存データに全対応・4本柱)

- **幸せに生きる**
  1. **資産(安定)** — 収支(PL/CF: 月収支・貯蓄) / 純資産(BS: 総資産×純資産アドバイザー)
  2. **心身(健康)**
     - 体: 睡眠 / 運動・心肺(VO2max) / 栄養(タンパク/水分/酒) / 体組成(FFMI/体脂肪) / 頭痛
     - 心: 気分・ストレス(主観チェックイン) / 内省・瞑想 / 意味・楽しみ(本/映画/漫画)
     - 依存: カフェイン・アルコール / スマホ(スクリーンタイム)
  3. **成長(仕事)** — 仕事の成果 / 学習・発話 / 目標管理
  4. **つながり(関係)** — ジャーナルの「話した人」テンプレから集計(新規データ源)

## つながりのデータ源(確定)

新テーブルや面倒な記録は作らず、**既存ジャーナルのテンプレに「話した人」欄**を足す。
週次で「誰と話した/会ったか(名前 or 人数)+ 満足度」を軽くタップ記録し、つながり領域の
達成度に使う。既存の journal 基盤を再利用。

## 表示原則: プログレッシブ・ディスクロージャー

- **ホーム = 4本柱カードのみ**。各カードは「一目の要約(1指標 + 状態色)」だけ。とことん少なく。
- カードをタップ → その柱のサブ領域一覧 → さらに指標 と**段階的に開く**。全解像度は掘れば出る。
- 既存の StatusLamps / トレンドカード / 各パネルは**下層に温存**(捨てない)。上層だけ静かにする。
- Apple Health 的: 余白・階層・1画面1目的。演出は初回/更新時のみ([[healthcare-ui-prefs]] 踏襲)。

## 実装方針(既存を尊重・段階的)

既存のライフドメイン基盤(LIFE_DOMAINS / 重み / ライフスコア / LifeSection)を**再マップ**する。
既存の健康・資産・学習・仕事・瞑想・スクリーンタイム等の計測とパネルはそのまま活かし、
「4本柱 → サブ領域 → 既存パネル」の階層に**束ね直す**。

段階:
1. つながり: journal テンプレに「話した人」+ 集計 + つながり達成度(小・独立で安全)
2. 4本柱マッピング: 既存ドメインを 資産/心身/成長/つながり に束ねる分類定義
3. ホーム UI: 4本柱カード + ドリルダウン(既存パネルを下層へ)

## デプロイ前提(重要)

本番 fp7 の auto-deploy が docker グループ権限で停止中。**この再設計を含む全変更は、
`sudo usermod -aG docker tsuyoshi` + 再起動(ユーザーの sudo 必須)で auto-deploy が復活してから
本番反映**される。実装は先行できるが、反映はこの修復待ち。

## 非目標 (YAGNI)

- 人間関係の重い CRM 的記録。ライフスコアの重み体系の作り直し(既存を再マップするだけ)。
- 既存パネルの機能削除(解像度は落とさない)。
